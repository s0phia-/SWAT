# NOTE: this file was originally pinned circa 2019 and largely could not
# install on modern Python (Python 3.12+ removed distutils, which old gym
# needs -- see setuptools note below; and several packages' own setup.py
# scripts reject modern Python outright). It has since been audited against
# an actual `grep` of every import in src/*.py: everything below is either
# genuinely used, or (torchfold) lazily-imported and only needed for one
# specific --actor_type/--critic_type. Every other 2019-era pin that wasn't
# actually imported anywhere (tensorflow, baselines, opencv, matplotlib,
# visdom, tensorboard, wandb-adjacent transitive deps, etc.) has been removed
# rather than fixed one broken build at a time.
#
# INSTALL IN TWO STEPS -- gym==0.13.1 is NOT listed below on purpose:
#   pip install --no-deps gym==0.13.1
#   pip install -r requirements.txt
# gym's own setup.py hard-pins scipy/six/pyglet<=1.3.2/cloudpickle~=1.2.0.
# Of those, gym actually imports `six` at runtime just to import gym.spaces/
# gym.utils.seeding (both of which we do use), so it's listed explicitly
# below even though it's really gym's dependency, not this repo's. pyglet
# and scipy are NOT needed -- gym only reaches for those in its classic_
# control/box2d rendering, which nothing here touches -- and cloudpickle
# needs to be >=3.0 here, conflicting with gym's declared ~=1.2.0 pin.
# Installing gym normally alongside this file's cloudpickle>=3.0 requirement
# is an unresolvable conflict: pip won't let both cloudpickle~=1.2.0 and
# >=3.0 be satisfied at once. --no-deps sidesteps it entirely by never
# asking pip to satisfy gym's stale pins in the first place.

# -- core training pipeline (main.py): validated working on Python 3.14 / arm64 --
setuptools  # must import before gym: shims distutils, removed in Python 3.12+
six  # gym==0.13.1 itself does `from six import integer_types` in gym/utils/seeding.py
mujoco  # replaces mujoco-py: mujoco-py needs an activation-key/license-server
        # flow MuJoCo dropped in 2021, and has no arm64 wheels
cloudpickle>=3.0  # >=3.0 required: older cloudpickle can't parse Python 3.12+ bytecode
torch
numpy
xmltodict==0.12.0
wandb  # training-curve logging; run `wandb login` once, or pass --disable_wandb

# -- only needed for --actor_type/--critic_type smp (ModularActor/ModularCritic's
# folded batching path); import is now lazy, so this is optional otherwise.
# torchfold is unmaintained (last touched ~2018) and may not build against a
# modern torch's C++ ABI -- untested this session.
git+https://github.com/CampagneLaboratory/torchfold@c2ac41b#egg=torchfold

# -- only needed for visualize.py (video generation); also needs the `ffmpeg`
# binary on PATH (e.g. `brew install ffmpeg`) -- that's a system tool, not a
# pip package, so it isn't listed here.
pillow>=6.2.2
